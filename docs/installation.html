Installation

Requirements

  • PHP 7.4+
  • Laravel 6.x
  • ext-dom — XML parsing
  • ext-zlib — PDF stream decompression

Via Composer (Path Repository)

If you have the package locally, add a path repository to your host app's composer.json:

{
    "repositories": [
        {
            "type": "path",
            "url": "../xfa-pdf"
        }
    ]
}

Then require the package:

composer require xfa/pdf

Auto-Discovery

The service provider and facade are registered automatically via Laravel's package auto-discovery. For manual registration:

// config/app.php

'providers' => [
    Xfa\Pdf\XfaPdfServiceProvider::class,
],

'aliases' => [
    'XfaPdf' => Xfa\Pdf\Facades\XfaPdf::class,
],

Publish Assets

# Publish configuration
php artisan vendor:publish --tag=xfa-pdf-config

# Publish views (optional, for customization)
php artisan vendor:publish --tag=xfa-pdf-views

# Publish migrations (optional, only for Web UI document storage)
php artisan vendor:publish --tag=xfa-pdf-migrations
php artisan migrate
Tip: Migrations are only needed for the Web UI (upload/list/manage documents). The PHP API and CLI work without any database.

Verify Installation

# Check the CLI command is registered
php artisan xfa-pdf:manage --help

# Check routes are registered
php artisan route:list --path=xfa-pdf
