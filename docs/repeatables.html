Repeatable Items

XFA forms support repeatable subforms — rows that can be dynamically added or removed.

How XFA Repeatables Work

In the original XFA PDF, repeatable subforms are defined with <occur max="-1"> in the template. JavaScript calls instanceManager.addInstance() / removeInstance() to manage rows. This package replicates that behavior in PHP.

<!-- Template defines the repeatable structure -->
<subform name="cpd" layout="lr-tb">
  <occur min="0" max="-1" initial="1"/>
  <field name="activityType"><ui><choiceList/></ui></field>
  <field name="description"><ui><textEdit/></ui></field>
  <field name="hours"><ui><numericEdit/></ui></field>
</subform>

The datasets hold the actual data instances:

<cpdList>
  <cpd>
    <activityType>Course</activityType>
    <description>Advanced PHP</description>
    <hours>16</hours>
  </cpd>
  <cpd>
    <activityType>Conference</activityType>
    <description>PHPConf 2024</description>
    <hours>8</hours>
  </cpd>
</cpdList>

PHP API

$manager = app('xfa-pdf');
$xfaPdf = $manager->load('form.pdf');

// Get all CPD items
$items = $manager->getRepeatableItems(
    $xfaPdf, 'Section_7_CPD', 'cpdList'
);
// [
//   ['activityType' => 'Course', 'description' => 'Advanced PHP', 'hours' => '16'],
//   ['activityType' => 'Conference', 'description' => 'PHPConf', 'hours' => '8'],
// ]

// Add a new item
$xfaPdf = $manager->addRepeatableItem(
    $xfaPdf, 'Section_7_CPD', 'cpdList',
    ['activityType' => 'Workshop', 'description' => 'Testing', 'hours' => '4']
);

// Remove item at index 0
$xfaPdf = $manager->removeRepeatableItem(
    $xfaPdf, 'Section_7_CPD', 'cpdList', 0
);

// Replace all items at once
$xfaPdf = $manager->setRepeatableItems(
    $xfaPdf, 'Section_7_CPD', 'cpdList', 'cpd',
    [
        ['activityType' => 'Course', 'description' => 'New', 'hours' => '20'],
        ['activityType' => 'Seminar', 'description' => 'Other', 'hours' => '8'],
    ]
);

// Save changes
$manager->saveXfaPdf($xfaPdf);

Web UI

In the edit page, repeatable sections show + (add) and − (remove) buttons. These send AJAX requests to /xfa-pdf/{id}/add-item and /xfa-pdf/{id}/remove-item. See Web UI Guide for details.
