Section 14: Additional Information

This section captures any additional information requested by the Responsible Officer. If specific information has been requested, the doctor provides details and supporting attachments.

Section name: Section_14_AdditionalInfo
Fluent accessor: XfaPdf::additionalInfo()
Navigation label: Additional information

What's in this section?

FieldTypeWhat it is
specificInfoRequestedRadio (yes/no)Has specific information been requested by the RO? (CONDITIONAL TRIGGER)
specificInfoDetailsMultiline textDetails of what was requested (shown when "yes")
additionalAttachments > attachRepeatableAdditional attachment rows
roleSelectType of supporting information
descriptionMultiline textDescription of the attachment
outcomeMultiline textOutcome or learning from the information
supportLocationSelectWhere the supporting evidence is held
firstEntryTextInitial entry reference
amendDataTextAmendment data reference
appraiserCommentsMultiline textAppraiser comments (mirrors to Section 19)
Conditional fields: The highlighted field above (specificInfoDetails) is only visible in the XFA form when specificInfoRequested is set to "yes". When set to "no", that field is hidden. In our package, you can always read and write it regardless of the radio value — the conditional visibility only applies within Adobe Reader.

The attach rows within additionalAttachments are repeatable — each row represents one piece of additional information or attachment. The remaining fields are standalone.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Additional Information
$data = XfaPdf::read('Section_14_AdditionalInfo');

// Or using the fluent accessor
$data = XfaPdf::additionalInfo()->read();

// $data looks like:
// [
//     'specificInfoRequested' => 'yes',
//     'specificInfoDetails' => 'Please provide evidence of CPD related to...',
//     'additionalAttachments' => [
//         'attach' => [
//             [
//                 'role' => 'CPD',
//                 'description' => 'Certificate from RCGP conference...',
//                 'outcome' => 'Updated knowledge on respiratory management...',
//                 'supportLocation' => 'Uploaded',
//                 'firstEntry' => '',
//                 'amendData' => '',
//             ],
//         ],
//     ],
//     'appraiserComments' => '',
// ]

Read a single field:

$requested = XfaPdf::additionalInfo()->readField('specificInfoRequested');
// 'yes' or 'no'

$details = XfaPdf::additionalInfo()->readField('specificInfoDetails');
// 'Please provide evidence of CPD related to...'

Radio / Conditional Fields

The specificInfoRequested field is a radio button group (yes/no) that controls the visibility of the specificInfoDetails textarea. In the XFA form, selecting "yes" reveals the details field; selecting "no" hides it.

Has specific information been requested?

Field: specificInfoRequested

This radio group asks whether the Responsible Officer has requested specific additional information from the doctor. Set it to "yes" or "no".

// Indicate that specific information has been requested
XfaPdf::additionalInfo()->updateField(
    'specificInfoRequested',
    'yes'
);

// Or indicate no information was requested
XfaPdf::additionalInfo()->updateField(
    'specificInfoRequested',
    'no'
);

Details of requested information

Field: specificInfoDetails

When the RO has requested specific information, describe what was requested and the doctor's response. This field is only visible in Adobe Reader when specificInfoRequested is set to "yes".

XfaPdf::additionalInfo()->updateField(
    'specificInfoDetails',
    'The Responsible Officer requested evidence of CPD activity related '
    . 'to safeguarding training completed in the last 12 months. A copy '
    . 'of the Level 3 safeguarding certificate has been attached below.'
);

Setting the radio and details together

When specific information has been requested, set the radio and the details field in a single call:

// When information HAS been requested, set the radio + details
XfaPdf::additionalInfo()->update([
    'specificInfoRequested' => 'yes',
    'specificInfoDetails'   => 'The RO requested evidence of safeguarding '
                             . 'training and a reflection on the clinical '
                             . 'incident reported in Q3 2024.',
]);

// When no information was requested, just set the radio
XfaPdf::additionalInfo()->update([
    'specificInfoRequested' => 'no',
]);
Important: Setting specificInfoRequested to "yes" in the datasets XML does not automatically make the specificInfoDetails field visible in the saved PDF — XFA visibility is controlled by JavaScript scripts within the template. However, when Adobe Reader opens the PDF and processes the XFA layer, the radio value will trigger the script and the conditional field will appear with whatever data you have written into it. Always populate specificInfoDetails when setting the radio to "yes".

Additional Attachments (Repeatable Table)

Each row in the additional attachments table represents one piece of supporting information or evidence. You can add multiple rows, remove them, or replace all of them. Each row contains six fields: role, description, outcome, supportLocation, firstEntry, and amendData.

Adding a new attachment row

To add a new row with all fields filled in:

$manager = app('xfa-pdf');
$xfaPdf = $manager->load(storage_path('app/xfa-pdf/appraisal.pdf'));

// Add an attachment row
$xfaPdf = $manager->addRepeatableItem(
    $xfaPdf,
    'Section_14_AdditionalInfo',       // section name
    'additionalAttachments',           // container name
    [
        'role'            => 'CPD',
        'description'     => 'Level 3 Safeguarding Children and Adults certificate '
                           . 'completed via e-Learning for Healthcare (e-LfH) on '
                           . '15 March 2024.',
        'outcome'         => 'Updated knowledge on recognising and responding to '
                           . 'safeguarding concerns. Applied learning to practice '
                           . 'safeguarding policy review.',
        'supportLocation' => 'Uploaded',
        'firstEntry'      => '',
        'amendData'       => '',
    ]
);

$manager->saveXfaPdf($xfaPdf);

Replacing all rows

If you want to clear existing rows and set new ones from scratch, use setRepeatableItems():

$xfaPdf = $manager->setRepeatableItems(
    $xfaPdf,
    'Section_14_AdditionalInfo',       // section
    'additionalAttachments',           // container
    'attach',                          // element name
    [
        [
            'role'            => 'CPD',
            'description'     => 'Level 3 Safeguarding certificate from e-LfH.',
            'outcome'         => 'Updated safeguarding knowledge applied to '
                               . 'practice policy review.',
            'supportLocation' => 'Uploaded',
            'firstEntry'      => '',
            'amendData'       => '',
        ],
        [
            'role'            => 'Audit',
            'description'     => 'Clinical incident reflection report requested '
                               . 'by RO following Q3 2024 serious event review.',
            'outcome'         => 'Identified system improvements for patient '
                               . 'handover processes. Changes implemented and '
                               . 'shared with practice team.',
            'supportLocation' => 'Uploaded',
            'firstEntry'      => '',
            'amendData'       => '',
        ],
    ]
);

$manager->saveXfaPdf($xfaPdf);

Reading existing rows

$rows = $manager->getRepeatableItems(
    $xfaPdf,
    'Section_14_AdditionalInfo',
    'additionalAttachments'
);

foreach ($rows as $index => $row) {
    echo "Row {$index}: [{$row['role']}] {$row['description']}\n";
}
// Row 0: [CPD] Level 3 Safeguarding certificate from e-LfH.
// Row 1: [Audit] Clinical incident reflection report requested by RO...

Removing a row

Remove a row by its index (0-based):

// Remove the second row (index 1)
$xfaPdf = $manager->removeRepeatableItem(
    $xfaPdf,
    'Section_14_AdditionalInfo',
    'additionalAttachments',
    1
);

$manager->saveXfaPdf($xfaPdf);

Text Fields

Appraiser's comments

Field: appraiserComments

Written by the appraiser, summarising their view of the additional information provided. This field mirrors its content to Section 19 (Appraisal Summary) so the appraiser's comments appear in both places.

XfaPdf::additionalInfo()->updateField(
    'appraiserComments',
    'The Responsible Officer requested evidence of safeguarding training '
    . 'and a reflection on the clinical incident from Q3 2024. Dr Smith '
    . 'has provided both items. The safeguarding certificate confirms '
    . 'completion of Level 3 training. The incident reflection is thorough '
    . 'and demonstrates appropriate learning and system improvement.'
);

Complete Controller Example

Here's a full example filling the entire Additional Information section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillAdditionalInfo(Request $request)
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        $manager = app('xfa-pdf');
        $xfaPdf = $manager->load($pdfPath);

        // -----------------------------------------------
        // 1. Set the radio and conditional details
        // -----------------------------------------------
        XfaPdf::setFile($pdfPath);

        $infoRequested = $request->input('info_requested', false);

        if ($infoRequested) {
            // RO has requested specific information
            XfaPdf::additionalInfo()->update([
                'specificInfoRequested' => 'yes',
                'specificInfoDetails'   => $request->input(
                    'info_details',
                    'The RO requested evidence of safeguarding training '
                    . 'and a reflection on the Q3 2024 clinical incident.'
                ),
            ]);
        } else {
            // No specific information requested
            XfaPdf::additionalInfo()->updateField(
                'specificInfoRequested',
                'no'
            );
        }

        // -----------------------------------------------
        // 2. Add attachment rows
        // -----------------------------------------------
        $xfaPdf = $manager->setRepeatableItems(
            $xfaPdf,
            'Section_14_AdditionalInfo',
            'additionalAttachments',
            'attach',
            [
                [
                    'role'            => 'CPD',
                    'description'     => 'Level 3 Safeguarding Children and '
                                       . 'Adults certificate. Completed via '
                                       . 'e-Learning for Healthcare on '
                                       . '15 March 2024.',
                    'outcome'         => 'Updated knowledge on recognising '
                                       . 'and responding to safeguarding '
                                       . 'concerns. Applied to practice '
                                       . 'safeguarding policy review.',
                    'supportLocation' => 'Uploaded',
                    'firstEntry'      => '',
                    'amendData'       => '',
                ],
                [
                    'role'            => 'Audit',
                    'description'     => 'Clinical incident reflection '
                                       . 'report: patient handover failure '
                                       . 'in Q3 2024. Structured reflection '
                                       . 'using Gibbs reflective cycle.',
                    'outcome'         => 'Identified system improvements '
                                       . 'for handover processes. New '
                                       . 'checklist implemented and shared '
                                       . 'with the practice team.',
                    'supportLocation' => 'Uploaded',
                    'firstEntry'      => '',
                    'amendData'       => '',
                ],
            ]
        );

        // -----------------------------------------------
        // 3. Appraiser comments
        // -----------------------------------------------
        XfaPdf::additionalInfo()->updateField(
            'appraiserComments',
            'The RO requested safeguarding evidence and an incident '
            . 'reflection. Dr Smith has provided both. The safeguarding '
            . 'certificate confirms Level 3 completion. The incident '
            . 'reflection is thorough and demonstrates appropriate '
            . 'learning with tangible system improvements.'
        );

        // -----------------------------------------------
        // 4. Save the PDF
        // -----------------------------------------------
        $manager->saveXfaPdf($xfaPdf);
        // Or: XfaPdf::save();

        return redirect()->back()->with(
            'success',
            'Additional Information saved.'
        );
    }

    /**
     * Read and display existing Additional Information data.
     */
    public function showAdditionalInfo()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        $manager = app('xfa-pdf');
        $xfaPdf = $manager->load($pdfPath);

        XfaPdf::setFile($pdfPath);

        $data = XfaPdf::additionalInfo()->read();

        // Check if conditional field should be shown
        $showDetails = ($data['specificInfoRequested'] ?? '') === 'yes';

        // Get attachment rows
        $attachments = $manager->getRepeatableItems(
            $xfaPdf,
            'Section_14_AdditionalInfo',
            'additionalAttachments'
        );

        return view('appraisal.additional-info', [
            'infoRequested'    => $showDetails,
            'infoDetails'      => $data['specificInfoDetails'] ?? '',
            'attachments'      => $attachments,
            'appraiserComments' => $data['appraiserComments'] ?? '',
        ]);
    }
}
