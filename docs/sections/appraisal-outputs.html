Section 20: Appraisal Outputs

The final section containing the formal appraisal outcome statements, declarations, and signatures from both the doctor and appraiser(s).

Section name: Section_20_AppraisalOutputs
Fluent accessor: XfaPdf::appraisalOutputs()
Navigation label: Appraisal outputs

There are no repeatable items or conditional fields in this section. The five statement fields are simple radio groups (not conditional triggers). The second appraiser fields are always present in the datasets XML; their visibility in the rendered form is controlled by the requireSecondAppraiser radio in Section 3.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Appraisal Outputs
$data = XfaPdf::read('Section_20_AppraisalOutputs');

// Or using the fluent accessor
$data = XfaPdf::appraisalOutputs()->read();

// $data looks like:
// [
//     'statementOne' => 'Agree',
//     'statementTwo' => 'Agree',
//     'statementThree' => 'Agree',
//     'statementFour' => 'Agree',
//     'statementFive' => 'Agree',
//     'appraiserComments' => 'No additional comments for the responsible officer.',
//     'appraiserIssues' => '',
//     'doctorResponse' => 'I agree with the appraiser\'s summary.',
//     'doctorChk' => '1',
//     'doctorFullName' => 'Dr Sarah Mitchell',
//     'doctorGMCNumber' => '1234567',
//     'appraiserOneChk' => '1',
//     'appraiserOneFullName' => 'Dr James Henderson',
//     'appraiserOneGMCNumber' => '7654321',
//     'appraiserTwoChk' => '',
//     'appraiserTwoFullName' => '',
//     'appraiserTwoGMCNumber' => '',
//     'dateOfMeeting' => '2025-01-15',
// ]

Read a single field:

$statement = XfaPdf::appraisalOutputs()->readField('statementOne');
// 'Agree'

$meetingDate = XfaPdf::appraisalOutputs()->readField('dateOfMeeting');
// '2025-01-15'

Statement Radio Fields

The five outcome statements are radio button groups with two options: "Agree" and "Disagree". These record the formal outcome of the appraisal discussion. Both the doctor and appraiser must agree or disagree with each statement.

Statement 1

Field: statementOne

The first formal outcome statement. Set to "Agree" or "Disagree".

XfaPdf::appraisalOutputs()->updateField(
    'statementOne',
    'Agree'
);

Statement 2

Field: statementTwo

XfaPdf::appraisalOutputs()->updateField(
    'statementTwo',
    'Agree'
);

Statement 3

Field: statementThree

XfaPdf::appraisalOutputs()->updateField(
    'statementThree',
    'Agree'
);

Statement 4

Field: statementFour

XfaPdf::appraisalOutputs()->updateField(
    'statementFour',
    'Agree'
);

Statement 5

Field: statementFive

XfaPdf::appraisalOutputs()->updateField(
    'statementFive',
    'Agree'
);

You can also set all five statements at once using update():

XfaPdf::appraisalOutputs()->update([
    'statementOne'   => 'Agree',
    'statementTwo'   => 'Agree',
    'statementThree' => 'Agree',
    'statementFour'  => 'Agree',
    'statementFive'  => 'Agree',
]);
Note: The value must be exactly "Agree" or "Disagree" (capitalised). These match the radio button options in the XFA template. An unrecognised value will still be written to the datasets XML but may not display correctly in Adobe Acrobat.

Text & Textarea Fields

These fields capture the appraiser's comments, any issues for the responsible officer, the doctor's response, and the identity details for both parties.

Appraiser comments

Field: appraiserComments

Comments from the appraiser to assist the responsible officer. This is a multiline textarea for any observations, recommendations, or context the appraiser wishes to provide.

XfaPdf::appraisalOutputs()->updateField(
    'appraiserComments',
    'No additional comments for the responsible officer. Dr Mitchell has '
    . 'provided comprehensive evidence across all domains and I am '
    . 'satisfied that she is meeting the requirements for revalidation. '
    . 'Her portfolio was well-organised and her reflections were '
    . 'thoughtful and demonstrated genuine learning.'
);

Appraiser issues

Field: appraiserIssues

Any other issues the appraiser wishes to bring to the attention of the responsible officer. This is typically left empty when there are no concerns.

XfaPdf::appraisalOutputs()->updateField(
    'appraiserIssues',
    ''
);

Doctor's response

Field: doctorResponse

The doctor's response to the appraiser's summary and comments. This gives the doctor an opportunity to add context or respond to any observations.

XfaPdf::appraisalOutputs()->updateField(
    'doctorResponse',
    'I agree with the appraiser\'s summary and am satisfied that the '
    . 'appraisal discussion was thorough and fair. The agreed PDP goals '
    . 'are appropriate and achievable within the coming appraisal year. '
    . 'I found the discussion about musculoskeletal medicine development '
    . 'particularly helpful in shaping my plans for the year ahead.'
);

Doctor's full name

Field: doctorFullName

The doctor's full name as it appears in the declaration.

XfaPdf::appraisalOutputs()->updateField(
    'doctorFullName',
    'Dr Sarah Mitchell'
);

Doctor's GMC number

Field: doctorGMCNumber

The doctor's GMC registration number in the declaration section.

XfaPdf::appraisalOutputs()->updateField(

Checkbox Fields

The declaration checkboxes confirm that the doctor and appraiser(s) agree with the formal declarations. Set to "1" for checked or "" (empty string) for unchecked.

Doctor's declaration

Field: doctorChk

Confirms the doctor agrees with the declaration statement. The declaration covers accuracy of information, fitness to practise, and consent for the appraisal output to be shared with the responsible officer.

XfaPdf::appraisalOutputs()->updateField(
    'doctorChk',
    '1'
);

First appraiser's declaration

Field: appraiserOneChk

Confirms the first appraiser agrees with the declaration statement.

XfaPdf::appraisalOutputs()->updateField(
    'appraiserOneChk',
    '1'
);

Second appraiser's declaration

Field: appraiserTwoChk

Confirms the second appraiser agrees with the declaration statement. Only relevant when a second appraiser is required.

// When a second appraiser is present
XfaPdf::appraisalOutputs()->updateField(
    'appraiserTwoChk',
    '1'
);

// When no second appraiser, leave unchecked
XfaPdf::appraisalOutputs()->updateField(
    'appraiserTwoChk',
    ''
);
Tip: Checkbox values in XFA are "1" for checked and "" (empty string) for unchecked. Do not use "true"/"false" or "yes"/"no" — these will be written to the XML but may not render correctly in Adobe Acrobat.

Date Field

Date of the appraisal meeting

Field: dateOfMeeting

The date when the appraisal meeting took place. Use YYYY-MM-DD format.

XfaPdf::appraisalOutputs()->updateField(
    'dateOfMeeting',
    '2025-01-15'
);
Note: Use the YYYY-MM-DD format for date fields. This matches the XFA date format specification date{YYYY-MM-DD} used in the template. Other formats may not display correctly.

Complete Controller Example

Here's a full example filling the entire Appraisal Outputs section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillAppraisalOutputs()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');

        // -----------------------------------------------
        // 1. Fill all output fields
        // -----------------------------------------------
        XfaPdf::setFile($pdfPath);

        XfaPdf::appraisalOutputs()->update([
            // Outcome statements
            'statementOne'   => 'Agree',
            'statementTwo'   => 'Agree',
            'statementThree' => 'Agree',
            'statementFour'  => 'Agree',
            'statementFive'  => 'Agree',

            // Appraiser comments
            'appraiserComments' =>
                'No additional comments for the responsible officer. '
                . 'Dr Mitchell has provided comprehensive evidence across '
                . 'all domains and I am satisfied that she is meeting the '
                . 'requirements for revalidation. Her portfolio was '
                . 'well-organised and her reflections were thoughtful.',

            'appraiserIssues' => '',

            // Doctor's response
            'doctorResponse' =>
                'I agree with the appraiser\'s summary and am satisfied '
                . 'that the appraisal discussion was thorough and fair. '
                . 'The agreed PDP goals are appropriate and achievable '
                . 'within the coming appraisal year.',

            // Doctor declaration
            'doctorChk'       => '1',
            'doctorFullName'  => 'Dr Sarah Mitchell',
            'doctorGMCNumber' => '1234567',

            // First appraiser declaration
            'appraiserOneChk'       => '1',
            'appraiserOneFullName'  => 'Dr James Henderson',
            'appraiserOneGMCNumber' => '7654321',

            // Second appraiser (leave empty if not required)
            'appraiserTwoChk'       => '',
            'appraiserTwoFullName'  => '',
            'appraiserTwoGMCNumber' => '',

            // Meeting date
            'dateOfMeeting' => '2025-01-15',
        ]);

        // -----------------------------------------------
        // 2. Save the PDF
        // -----------------------------------------------
        XfaPdf::save();

        return redirect()->back()->with('success', 'Appraisal Outputs saved.');
    }
}
