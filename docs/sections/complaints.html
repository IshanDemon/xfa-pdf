Section 11: Review of Complaints and Compliments

This section records whether the doctor has been named in any complaints, the learning from those complaints, and any compliments received.

Section name: Section_11_Complaints
Fluent accessor: XfaPdf::complaints()

What's in this section?

FieldTypeWhat it is
haveBeenNamedRadio (not/have)Has the doctor been named in any complaints?
complaintLearningMultiline textLearning arising from complaints (conditional — shown when "have")
ComplimentEvents > complimentDetalsRepeatableComplaint/compliment event rows (each with role, description, outcome, etc.)
complimentsMultiline textCompliments received from patients, colleagues, or organisations
appraiserCommentsMultiline textAppraiser's comments on complaints and compliments (mirrors to Section 19)
Conditional field: The haveBeenNamed radio field uses values "not" and "have" (not true/false). When set to "have", the complaintLearning textarea becomes visible. When set to "not", it is hidden.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Complaints
$data = XfaPdf::read('Section_11_Complaints');

// Or using the fluent accessor
$data = XfaPdf::complaints()->read();

// $data looks like:
// [
//     'haveBeenNamed' => 'have',
//     'complaintLearning' => 'The complaint highlighted the importance of...',
//     'ComplimentEvents' => [
//         'complimentDetals' => [
//             [
//                 'role' => 'GP',
//                 'description' => 'Patient complaint regarding...',
//                 'outcome' => 'Resolved through mediation...',
//                 'supportLocation' => 'Practice',
//                 'firstEntry' => '2024-03-15',
//                 'amendData' => '',
//             ],
//         ],
//     ],
//     'compliments' => 'Received thank-you card from patient...',
//     'appraiserComments' => '',
// ]

Read a single field:

$named = XfaPdf::complaints()->readField('haveBeenNamed');
// 'have' or 'not'

Radio / Conditional Fields

The haveBeenNamed field is a radio group that controls whether the complaintLearning textarea is displayed. Set it to "have" or "not".

Setting the radio value

Field: haveBeenNamed

When the doctor has been named in a complaint, set this to "have". This will cause Adobe Acrobat to show the complaintLearning field.

// Doctor HAS been named in a complaint
XfaPdf::complaints()->updateField('haveBeenNamed', 'have');

// Doctor has NOT been named in a complaint
XfaPdf::complaints()->updateField('haveBeenNamed', 'not');

Complaint learning

Field: complaintLearning

This textarea is only visible in the rendered PDF when haveBeenNamed is set to "have". It captures the doctor's reflection on what they learned from the complaint(s).

XfaPdf::complaints()->updateField(
    'complaintLearning',
    'The complaint related to a delay in referral for a patient with '
    . 'worsening symptoms. On reflection, I have updated my practice '
    . 'to include a safety-netting protocol for patients presenting '
    . 'with persistent or escalating symptoms. I now use a structured '
    . 'follow-up checklist and have discussed the case with my '
    . 'appraisal group. The complaint was resolved through the '
    . 'practice complaints procedure and the patient was satisfied '
    . 'with the outcome.'
);
Important: Always set haveBeenNamed to "have" before or alongside populating complaintLearning. The data will be saved either way, but Adobe Acrobat will only display the learning field when the radio is set to "have".

Complaint/Compliment Events (Repeatable Table)

Each row in the complimentDetals table represents one complaint or compliment event. The rows live inside the ComplimentEvents container. Each row has these fields:

Row fieldTypeWhat it is
roleSelectThe doctor's role at the time of the event
descriptionMultiline textDescription of the complaint or compliment
outcomeMultiline textOutcome and any actions taken
supportLocationSelectWhere support was accessed (e.g. Practice, Trust, NHS England)
firstEntryTextDate or note when the event was first recorded
amendDataTextDate or note of any amendments to the record

Adding a complaint/compliment event

To add a new event row:

$manager = app('xfa-pdf');
$xfaPdf = $manager->load(storage_path('app/xfa-pdf/appraisal.pdf'));

// Add a complaint event
$xfaPdf = $manager->addRepeatableItem(
    $xfaPdf,
    'Section_11_Complaints',      // section name
    'ComplimentEvents',            // container name
    [
        'role'            => 'GP Partner',
        'description'     => 'Formal complaint received regarding a delay in onward '
                           . 'referral for a patient with progressive knee pain. The '
                           . 'patient felt their symptoms were not taken seriously at '
                           . 'the initial consultation.',
        'outcome'         => 'The complaint was investigated under the practice complaints '
                           . 'procedure. I reviewed the consultation notes and acknowledged '
                           . 'that an earlier referral would have been appropriate. The '
                           . 'patient received an apology and was referred to orthopaedics. '
                           . 'No further action was taken.',
        'supportLocation' => 'Practice',
        'firstEntry'      => '2024-06-12',
        'amendData'       => '',
    ]
);

$manager->saveXfaPdf($xfaPdf);

Replacing all event rows

If you want to clear existing rows and set new ones from scratch, use setRepeatableItems():

$xfaPdf = $manager->setRepeatableItems(
    $xfaPdf,
    'Section_11_Complaints',      // section
    'ComplimentEvents',            // container
    'complimentDetals',            // element name
    [
        [
            'role'            => 'GP Partner',
            'description'     => 'Complaint regarding delay in referral for progressive '
                               . 'knee pain. Patient felt symptoms were not taken seriously.',
            'outcome'         => 'Investigated under practice complaints procedure. '
                               . 'Acknowledged earlier referral was appropriate. Patient '
                               . 'received apology and orthopaedic referral.',
            'supportLocation' => 'Practice',
            'firstEntry'      => '2024-06-12',
            'amendData'       => '',
        ],
        [
            'role'            => 'GP Partner',
            'description'     => 'Compliment received from patient with chronic COPD. '
                               . 'Patient praised the continuity of care and the '
                               . 'comprehensive review of their inhaler technique.',
            'outcome'         => 'Positive feedback shared with the practice team. '
                               . 'Used as an example in the practice newsletter.',
            'supportLocation' => 'Practice',
            'firstEntry'      => '2024-09-03',
            'amendData'       => '',
        ],
    ]
);

$manager->saveXfaPdf($xfaPdf);

Reading existing event rows

$rows = $manager->getRepeatableItems(
    $xfaPdf,
    'Section_11_Complaints',
    'ComplimentEvents'
);

foreach ($rows as $index => $row) {
    echo "Event {$index}: {$row['role']} - {$row['description']}\n";
}
// Event 0: GP Partner - Complaint regarding delay in referral...
// Event 1: GP Partner - Compliment received from patient with chronic COPD...

Removing an event row

Remove a row by its index (0-based):

// Remove the first event row (index 0)
$xfaPdf = $manager->removeRepeatableItem(
    $xfaPdf,
    'Section_11_Complaints',
    'ComplimentEvents',
    0
);

$manager->saveXfaPdf($xfaPdf);

Text Fields

Below the event table, there are two standalone text fields for compliments and appraiser comments.

Compliments received

Field: compliments

A free-text area for recording compliments from patients, colleagues, or organisations.

XfaPdf::complaints()->updateField(
    'compliments',
    'Received a thank-you card from a patient with newly diagnosed diabetes '
    . 'who appreciated the time taken to explain their condition and treatment '
    . 'options. Also received positive feedback from the district nursing team '
    . 'regarding the quality of discharge summaries and communication around '
    . 'shared patients. A colleague commended my contribution to the practice '
    . 'significant event audit meeting in September 2024.'
);

Appraiser's comments

Field: appraiserComments

Written by the appraiser, summarising their view of the doctor's complaints and compliments. This field mirrors into Section 19 (Appraisal Summary).

XfaPdf::complaints()->updateField(
    'appraiserComments',
    'Dr Smith has been open and reflective about the complaint received '
    . 'this year. The learning identified is appropriate and the changes '
    . 'to practice (safety-netting protocol and structured follow-up) are '
    . 'proportionate and well-considered. The compliments demonstrate good '
    . 'patient rapport and effective communication with the wider team. '
    . 'I am satisfied that complaints are being used as a learning opportunity.'
);

Complete Controller Example

Here's a full example filling the entire Complaints section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillComplaints()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        $manager = app('xfa-pdf');
        $xfaPdf = $manager->load($pdfPath);

        // -----------------------------------------------
        // 1. Set the radio: has the doctor been named?
        // -----------------------------------------------
        XfaPdf::setFile($pdfPath);
        XfaPdf::complaints()->updateField('haveBeenNamed', 'have');

        // -----------------------------------------------
        // 2. Fill complaint learning (shown because radio = "have")
        // -----------------------------------------------
        XfaPdf::complaints()->updateField(
            'complaintLearning',
            'The complaint related to a delay in referral for a patient '
            . 'with worsening knee pain. I have reflected on this and '
            . 'introduced a safety-netting protocol for patients with '
            . 'persistent or escalating symptoms. I now use a structured '
            . 'follow-up checklist and have discussed the case at our '
            . 'practice significant event meeting.'
        );

        // -----------------------------------------------
        // 3. Add complaint/compliment event rows
        // -----------------------------------------------
        $xfaPdf = $manager->setRepeatableItems(
            $xfaPdf,
            'Section_11_Complaints',
            'ComplimentEvents',
            'complimentDetals',
            [
                [
                    'role'            => 'GP Partner',
                    'description'     => 'Formal complaint from patient regarding '
                                       . 'delay in orthopaedic referral for '
                                       . 'progressive knee pain.',
                    'outcome'         => 'Investigated under practice complaints '
                                       . 'procedure. Earlier referral acknowledged '
                                       . 'as appropriate. Patient received apology '
                                       . 'and was referred to orthopaedics.',
                    'supportLocation' => 'Practice',
                    'firstEntry'      => '2024-06-12',
                    'amendData'       => '',
                ],
                [
                    'role'            => 'GP Partner',
                    'description'     => 'Compliment from patient with chronic COPD '
                                       . 'praising continuity of care and thorough '
                                       . 'inhaler technique review.',
                    'outcome'         => 'Positive feedback shared with practice '
                                       . 'team and included in the practice '
                                       . 'newsletter.',
                    'supportLocation' => 'Practice',
                    'firstEntry'      => '2024-09-03',
                    'amendData'       => '',
                ],
                [
                    'role'            => 'Clinical Lead - PCN',
                    'description'     => 'Letter of thanks from North Manchester '
                                       . 'PCN board recognising leadership of the '
                                       . 'respiratory pathway redesign project.',
                    'outcome'         => 'Shared with appraiser as supporting '
                                       . 'information for this appraisal.',
                    'supportLocation' => 'Practice',
                    'firstEntry'      => '2024-11-20',
                    'amendData'       => '',
                ],
            ]
        );

        // -----------------------------------------------
        // 4. Fill compliments text
        // -----------------------------------------------
        XfaPdf::complaints()->updateField(
            'compliments',
            'Received a thank-you card from a newly diagnosed diabetes '
            . 'patient. Positive feedback from the district nursing team '
            . 'on discharge summaries. Letter of thanks from PCN board '
            . 'for respiratory pathway work.'
        );

        // -----------------------------------------------
        // 5. Appraiser's comments
        // -----------------------------------------------
        XfaPdf::complaints()->updateField(
            'appraiserComments',
            'Dr Smith has been open and reflective about the complaint '
            . 'received this year. The learning identified is appropriate '
            . 'and changes to practice are proportionate. Compliments '
            . 'demonstrate good patient rapport and team communication. '
            . 'I am satisfied that complaints are used as learning '
            . 'opportunities.'
        );

        // -----------------------------------------------
        // 6. Save the PDF
        // -----------------------------------------------
        $manager->saveXfaPdf($xfaPdf);
        // Or: XfaPdf::save();

        return redirect()->back()->with('success', 'Complaints section saved.');
    }
}
