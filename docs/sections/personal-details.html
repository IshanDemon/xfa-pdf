Section 3: Personal Details

This section captures the doctor's personal and contact information, GMC number, designated body, medical qualifications, appraisal year, and appraiser details.

Section name: Section_3_AppraisalDetails
Fluent accessor: XfaPdf::personalDetails()

What's in this section?

FieldTypeWhat it is
appraiseeNameTextareaFull name of the doctor
GMCNumberTextareaDoctor's GMC registration number
contactAddressTextareaContact address for correspondence
contactPhoneTextareaContact telephone number
contactEmailTextareaContact email address
designatedBodyNameTextName of the designated body
medicalQualificationsRepeatableContains qualification, awardingBody, year
appraisalYearSelect (dropdown)Year of this appraisal (2016/17 through 2036/37)
nextAppraisalDueDateDue date of next revalidation recommendation
appraiserNameTextName of the appraiser
requireSecondAppraiserRadio (yes/no)Whether a second appraiser is required (conditional trigger)
secondAppraiserNameTextareaName of second appraiser (visible when "yes")
secondAppraiserGMCNumberTextGMC number of second appraiser (visible when "yes")
Conditional fields: The requireSecondAppraiser radio is a conditional trigger. When set to "yes", the fields secondAppraiserName and secondAppraiserGMCNumber become visible in the XFA form. Your code should still set these values regardless of visibility — the XFA rendering engine controls what the user sees.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Personal Details
$data = XfaPdf::read('Section_3_AppraisalDetails');

// Or using the fluent accessor
$data = XfaPdf::personalDetails()->read();

// $data looks like:
// [
//     'appraiseeName' => 'Dr Sarah Mitchell',
//     'GMCNumber' => '1234567',
//     'contactAddress' => '42 Riverside Drive, Manchester M1 2AB',
//     'contactPhone' => '0161 234 5678',
//     'contactEmail' => 'sarah.mitchell@riverside-mc.nhs.uk',
//     'designatedBodyName' => 'NHS England (North West)',
//     'medicalQualifications' => [
//         'qualRpt' => [
//             ['qualification' => 'MBBS', 'awardingBody' => 'University of Manchester', 'year' => '2005'],
//             ['qualification' => 'MRCGP', 'awardingBody' => 'Royal College of GPs', 'year' => '2009'],
//         ],
//     ],
//     'appraisalYear' => '2025/26',
//     'nextAppraisalDue' => '2026-09-30',
//     'appraiserName' => 'Dr James Henderson',
//     'requireSecondAppraiser' => 'no',
//     'secondAppraiserName' => '',
//     'secondAppraiserGMCNumber' => '',
// ]

Read a single field:

$name = XfaPdf::personalDetails()->readField('appraiseeName');
// 'Dr Sarah Mitchell'

$gmc = XfaPdf::personalDetails()->readField('GMCNumber');
// '1234567'

Medical Qualifications (Repeatable Table)

The medical qualifications table is a repeatable section where each row represents one qualification. Each row contains three fields: qualification (the degree or certificate name), awardingBody (the institution), and year (when it was awarded).

Adding a new qualification

To add a new qualification row:

$manager = app('xfa-pdf');
$xfaPdf = $manager->load(storage_path('app/xfa-pdf/appraisal.pdf'));

// Add a qualification row
$xfaPdf = $manager->addRepeatableItem(
    $xfaPdf,
    'Section_3_AppraisalDetails',     // section name
    'medicalQualifications',          // container name
    [
        'qualification' => 'MBBS',
        'awardingBody'  => 'University of Manchester',
        'year'          => '2005',
    ]
);

$manager->saveXfaPdf($xfaPdf);

Replacing all qualifications

If you want to clear existing qualifications and set new ones from scratch, use setRepeatableItems():

$xfaPdf = $manager->setRepeatableItems(
    $xfaPdf,
    'Section_3_AppraisalDetails',     // section
    'medicalQualifications',          // container
    'qualRpt',                        // element name
    [
        [
            'qualification' => 'MBBS',
            'awardingBody'  => 'University of Manchester',
            'year'          => '2005',
        ],
        [
            'qualification' => 'MRCGP',
            'awardingBody'  => 'Royal College of General Practitioners',
            'year'          => '2009',
        ],
        [
            'qualification' => 'DRCOG',
            'awardingBody'  => 'Royal College of Obstetricians and Gynaecologists',
            'year'          => '2007',
        ],
        [
            'qualification' => 'DipDerm',
            'awardingBody'  => 'Cardiff University',
            'year'          => '2012',
        ],
    ]
);

$manager->saveXfaPdf($xfaPdf);

Reading existing qualifications

$quals = $manager->getRepeatableItems(
    $xfaPdf,
    'Section_3_AppraisalDetails',
    'medicalQualifications'
);

foreach ($quals as $index => $qual) {
    echo "Row {$index}: {$qual['qualification']} — {$qual['awardingBody']} ({$qual['year']})\n";
}
// Row 0: MBBS — University of Manchester (2005)
// Row 1: MRCGP — Royal College of General Practitioners (2009)
// Row 2: DRCOG — Royal College of Obstetricians and Gynaecologists (2007)

Removing a qualification

Remove a qualification by its index (0-based):

// Remove the third qualification (index 2)
$xfaPdf = $manager->removeRepeatableItem(
    $xfaPdf,
    'Section_3_AppraisalDetails',
    'medicalQualifications',
    2
);

$manager->saveXfaPdf($xfaPdf);

Text Fields

These are the main identity and contact fields for the doctor. Each is a standalone text or textarea field.

Doctor's name

Field: appraiseeName

The full name of the doctor being appraised. This typically includes title, first name, and surname.

XfaPdf::personalDetails()->updateField(
    'appraiseeName',
    'Dr Sarah Mitchell'
);

GMC number

Field: GMCNumber

The doctor's General Medical Council registration number. This is a 7-digit number.

XfaPdf::personalDetails()->updateField(
    'GMCNumber',
    '1234567'
);

Contact address

Field: contactAddress

The doctor's correspondence address. This is a multiline textarea so you can include the full postal address.

XfaPdf::personalDetails()->updateField(
    'contactAddress',
    "Riverside Medical Centre\n42 High Street\nManchester\nM1 2AB"
);

Contact telephone

Field: contactPhone

The doctor's contact telephone number.

XfaPdf::personalDetails()->updateField(
    'contactPhone',
    '0161 234 5678'
);

Contact email

Field: contactEmail

The doctor's email address for appraisal-related correspondence.

XfaPdf::personalDetails()->updateField(
    'contactEmail',
    'sarah.mitchell@riverside-mc.nhs.uk'
);

Designated body

Field: designatedBodyName

The name of the designated body (the organisation responsible for the doctor's revalidation).

XfaPdf::personalDetails()->updateField(
    'designatedBodyName',
    'NHS England (North West)'
);

Select & Date Fields

These fields use dropdown selects and date pickers in the XFA form. When writing values programmatically, pass the value exactly as the dropdown option or date format expects.

Appraisal year

Field: appraisalYear

A dropdown select field with options ranging from 2016/17 through 2036/37. Pass the value as a string matching the dropdown option exactly.

XfaPdf::personalDetails()->updateField(
    'appraisalYear',
    '2025/26'
);
Note: The value must match one of the dropdown options exactly (e.g. '2025/26', not '2025-26' or '2025'). An unrecognised value will still be written to the datasets XML but may not display correctly in Adobe Acrobat.

Next revalidation due date

Field: nextAppraisalDue

A date field for when the next revalidation recommendation is due. Use YYYY-MM-DD format.

XfaPdf::personalDetails()->updateField(
    'nextAppraisalDue',
    '2026-09-30'
);

Appraiser's name

Field: appraiserName

The name of the person conducting the appraisal.

XfaPdf::personalDetails()->updateField(
    'appraiserName',
    'Dr James Henderson'
);

Radio & Conditional Fields

The requireSecondAppraiser field is a yes/no radio group that acts as a conditional trigger. When set to "yes", the XFA form reveals two additional fields for the second appraiser's details.

Second appraiser required?

Field: requireSecondAppraiser

A radio button group with values "yes" or "no". Setting this to "yes" makes the second appraiser fields visible in the rendered XFA form.

// No second appraiser needed
XfaPdf::personalDetails()->updateField(
    'requireSecondAppraiser',
    'no'
);

When a second appraiser is required, set the radio to "yes" and fill in the conditional fields:

// Second appraiser is required
XfaPdf::personalDetails()->updateField(
    'requireSecondAppraiser',
    'yes'
);

// Fill in the conditional fields
XfaPdf::personalDetails()->updateField(
    'secondAppraiserName',
    'Dr Patricia Clarke'
);

XfaPdf::personalDetails()->updateField(
    'secondAppraiserGMCNumber',
    '7654321'
);
Tip: Even when requireSecondAppraiser is set to "no", you can still write values to secondAppraiserName and secondAppraiserGMCNumber. The data will be stored in the datasets XML but the fields will remain hidden in the rendered form until the radio is changed to "yes".

Complete Controller Example

Here's a full example filling the entire Personal Details section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillPersonalDetails()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        $manager = app('xfa-pdf');
        $xfaPdf = $manager->load($pdfPath);

        // -----------------------------------------------
        // 1. Set medical qualifications (repeatable rows)
        // -----------------------------------------------
        $xfaPdf = $manager->setRepeatableItems(
            $xfaPdf,
            'Section_3_AppraisalDetails',
            'medicalQualifications',
            'qualRpt',
            [
                [
                    'qualification' => 'MBBS',
                    'awardingBody'  => 'University of Manchester',
                    'year'          => '2005',
                ],
                [
                    'qualification' => 'MRCGP',
                    'awardingBody'  => 'Royal College of General Practitioners',
                    'year'          => '2009',
                ],
                [
                    'qualification' => 'DRCOG',
                    'awardingBody'  => 'Royal College of Obstetricians and Gynaecologists',
                    'year'          => '2007',
                ],
            ]
        );

        // -----------------------------------------------
        // 2. Fill all text, select, date, and radio fields
        // -----------------------------------------------
        XfaPdf::setFile($pdfPath);

        XfaPdf::personalDetails()->update([
            // Identity
            'appraiseeName'      => 'Dr Sarah Mitchell',
            'GMCNumber'          => '1234567',

            // Contact details
            'contactAddress'     => "Riverside Medical Centre\n42 High Street\nManchester\nM1 2AB",
            'contactPhone'       => '0161 234 5678',
            'contactEmail'       => 'sarah.mitchell@riverside-mc.nhs.uk',

            // Designated body
            'designatedBodyName' => 'NHS England (North West)',

            // Appraisal details
            'appraisalYear'      => '2025/26',
            'nextAppraisalDue'   => '2026-09-30',
            'appraiserName'      => 'Dr James Henderson',

            // Second appraiser (conditional)
            'requireSecondAppraiser'  => 'no',
            'secondAppraiserName'     => '',
            'secondAppraiserGMCNumber' => '',
        ]);

        // -----------------------------------------------
        // 3. Save the PDF
        // -----------------------------------------------
        $manager->saveXfaPdf($xfaPdf);
        // Or: XfaPdf::save();

        return redirect()->back()->with('success', 'Personal Details saved.');
    }
}
