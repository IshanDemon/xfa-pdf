Section 5: Previous Appraisals

Records details of previous appraisals including the date, appraiser name, responsible officer, and designated body. If any details have changed since the last appraisal, the doctor can provide updated information.

Section name: Section_5_PreviousAppraisals
Fluent accessor: XfaPdf::previousAppraisals()
Navigation label: Record of annual appraisals

What's in this section?

FieldTypeWhat it is
dateOfLastAppraisalDateDate of last appraisal
appraisalDetailsChangeRdoRadio (yes/no)Have any details changed since last appraisal?
nameOfLastAppraiserMultiline textName of appraiser at last appraisal (conditional)
nameOfResponsibleOfficerMultiline textName of responsible officer (conditional)
nameOfDesignatedBodyMultiline textName of designated body (conditional)
attachmentIDTextInternal attachment reference

Conditional fields: The three highlighted fields above (nameOfLastAppraiser, nameOfResponsibleOfficer, nameOfDesignatedBody) are only visible in the XFA form when appraisalDetailsChangeRdo is set to "yes". When set to "no", those fields are hidden. In our package, you can always read and write them regardless of the radio value — the conditional visibility only applies within Adobe Reader.

This section has no repeatable items. All fields are standalone.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Previous Appraisals
$data = XfaPdf::read('Section_5_PreviousAppraisals');

// Or using the fluent accessor
$data = XfaPdf::previousAppraisals()->read();

// $data looks like:
// [
//     'dateOfLastAppraisal' => '2024-09-15',
//     'appraisalDetailsChangeRdo' => 'yes',
//     'nameOfLastAppraiser' => 'Dr. James Patterson',
//     'nameOfResponsibleOfficer' => 'Dr. Sarah Mitchell',
//     'nameOfDesignatedBody' => 'NHS England',
//     'attachmentID' => '',
//     'fileName' => '',
// ]

Read a single field:

$date = XfaPdf::previousAppraisals()->readField('dateOfLastAppraisal');
// '2024-09-15'

$changed = XfaPdf::previousAppraisals()->readField('appraisalDetailsChangeRdo');
// 'yes' or 'no'

Date Fields

Date of last appraisal

Field: dateOfLastAppraisal

The date when the doctor's most recent appraisal took place. The XFA form uses a dateTimeEdit control with a YYYY-MM-DD format pattern.

// Set the date of last appraisal (use YYYY-MM-DD format)
XfaPdf::previousAppraisals()->updateField(
    'dateOfLastAppraisal',
    '2024-09-15'
);

// You can also use Carbon to format the date
use Carbon\Carbon;

$lastAppraisalDate = Carbon::parse('15 September 2024');
XfaPdf::previousAppraisals()->updateField(
    'dateOfLastAppraisal',
    $lastAppraisalDate->format('Y-m-d')
);
Date format: Always use YYYY-MM-DD (ISO 8601) when writing date values. The XFA template defines its display format via the <picture> element, so Adobe Reader will format it appropriately for display. Passing dates in other formats (e.g. DD/MM/YYYY) may cause parsing errors in the XFA renderer.

Radio / Conditional Fields

The appraisalDetailsChangeRdo field is a radio button group (yes/no) that controls the visibility of three dependent fields. In the XFA form, selecting "yes" reveals the fields for entering changed details; selecting "no" hides them.

Have details changed?

Field: appraisalDetailsChangeRdo

This radio group asks whether the appraiser, responsible officer, or designated body has changed since the last appraisal. Set it to "yes" or "no".

// Indicate that details have changed
XfaPdf::previousAppraisals()->updateField(
    'appraisalDetailsChangeRdo',
    'yes'
);

// Or indicate no changes
XfaPdf::previousAppraisals()->updateField(
    'appraisalDetailsChangeRdo',
    'no'
);
Important: Setting appraisalDetailsChangeRdo to "yes" in the datasets XML does not automatically make the conditional fields visible in the saved PDF — XFA visibility is controlled by JavaScript scripts within the template. However, when Adobe Reader opens the PDF and processes the XFA layer, the radio value will trigger the script and the conditional fields will appear with whatever data you have written into them. Always populate the conditional fields when setting the radio to "yes".

Setting the radio and conditional fields together

When details have changed, set the radio and all three dependent fields in a single call:

// When details HAVE changed, set the radio + all conditional fields
XfaPdf::previousAppraisals()->update([
    'appraisalDetailsChangeRdo' => 'yes',
    'nameOfLastAppraiser'       => 'Dr. James Patterson',
    'nameOfResponsibleOfficer'  => 'Dr. Sarah Mitchell',
    'nameOfDesignatedBody'      => 'NHS England',
]);

// When details have NOT changed, just set the radio
XfaPdf::previousAppraisals()->update([
    'appraisalDetailsChangeRdo' => 'no',
]);

Text Fields

These three fields are conditional — they are only visible in Adobe Reader when appraisalDetailsChangeRdo is set to "yes". They allow the doctor to record updated details about their previous appraisal if anything has changed.

Name of last appraiser

Field: nameOfLastAppraiser

The name of the appraiser who conducted the doctor's most recent appraisal. This is a multiline text field, so you can include additional context such as their role or organisation.

XfaPdf::previousAppraisals()->updateField(
    'nameOfLastAppraiser',
    'Dr. James Patterson'
);

// With additional context
XfaPdf::previousAppraisals()->updateField(
    'nameOfLastAppraiser',
    "Dr. James Patterson\nGP Partner, Oakwood Surgery\nManchester"
);

Name of responsible officer

Field: nameOfResponsibleOfficer

The responsible officer (RO) overseeing the doctor's revalidation. This is typically a senior medical professional within the designated body.

XfaPdf::previousAppraisals()->updateField(
    'nameOfResponsibleOfficer',
    'Dr. Sarah Mitchell'
);

// With role details
XfaPdf::previousAppraisals()->updateField(
    'nameOfResponsibleOfficer',
    "Dr. Sarah Mitchell\nResponsible Officer, NHS England (North West)"
);

Name of designated body

Field: nameOfDesignatedBody

The designated body with which the doctor holds a prescribed connection for revalidation purposes. This is typically an NHS trust, CCG, or NHS England regional team.

XfaPdf::previousAppraisals()->updateField(
    'nameOfDesignatedBody',
    'NHS England'
);

// With full details
XfaPdf::previousAppraisals()->updateField(
    'nameOfDesignatedBody',
    "NHS England (North West Region)\nPiccadilly Place, Manchester M1 3BN"
);

Complete Controller Example

Here's a full example filling the entire Previous Appraisals section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillPreviousAppraisals(Request $request)
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        XfaPdf::setFile($pdfPath);

        // -----------------------------------------------
        // 1. Set the date of last appraisal
        // -----------------------------------------------
        XfaPdf::previousAppraisals()->updateField(
            'dateOfLastAppraisal',
            '2024-09-15'
        );

        // -----------------------------------------------
        // 2. Check if details have changed and fill
        //    conditional fields accordingly
        // -----------------------------------------------
        $detailsChanged = $request->input('details_changed', false);

        if ($detailsChanged) {
            // Details have changed: set radio to "yes"
            // and populate all three conditional fields
            XfaPdf::previousAppraisals()->update([
                'appraisalDetailsChangeRdo' => 'yes',
                'nameOfLastAppraiser'       => $request->input(
                    'last_appraiser',
                    'Dr. James Patterson'
                ),
                'nameOfResponsibleOfficer'  => $request->input(
                    'responsible_officer',
                    'Dr. Sarah Mitchell'
                ),
                'nameOfDesignatedBody'      => $request->input(
                    'designated_body',
                    'NHS England'
                ),
            ]);
        } else {
            // No changes: just set the radio to "no"
            XfaPdf::previousAppraisals()->updateField(
                'appraisalDetailsChangeRdo',
                'no'
            );
        }

        // -----------------------------------------------
        // 3. Save the PDF
        // -----------------------------------------------
        XfaPdf::save();

        return redirect()->back()->with(
            'success',
            'Previous Appraisals saved.'
        );
    }

    /**
     * Read and display existing Previous Appraisals data.
     */
    public function showPreviousAppraisals()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        XfaPdf::setFile($pdfPath);

        $data = XfaPdf::previousAppraisals()->read();

        // Check if conditional fields should be shown
        $showChangedDetails = ($data['appraisalDetailsChangeRdo'] ?? '') === 'yes';

        return view('appraisal.previous-appraisals', [
            'dateOfLastAppraisal'      => $data['dateOfLastAppraisal'] ?? '',
            'detailsChanged'           => $showChangedDetails,
            'nameOfLastAppraiser'      => $data['nameOfLastAppraiser'] ?? '',
            'nameOfResponsibleOfficer' => $data['nameOfResponsibleOfficer'] ?? '',
            'nameOfDesignatedBody'     => $data['nameOfDesignatedBody'] ?? '',
        ]);
    }
}
