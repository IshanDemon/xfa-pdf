Section 13: Probity and Health

This section captures the doctor's probity and health declarations. The doctor confirms they have no outstanding probity or health issues, or declares any concerns. It includes checkbox confirmations and a conditional text area that appears when the doctor has something to declare.

Section name: Section_13_Probity
Fluent accessor: XfaPdf::probity()
Navigation label: Probity and health statements

What's in this section?

FieldTypeWhat it is
probityChkCheckboxProbity confirmation checkbox (value "0" or "1")
anythingToDeclareRadio (nothing/something)Anything to declare? (CONDITIONAL TRIGGER)
suspendedOrRestrictedTextareaDeclaration details if suspended/restricted (conditional)
healthChkCheckboxHealth confirmation checkbox (value "0" or "1")
commentsToAppraiserTextareaAdditional comments to appraiser
Conditional fields: The highlighted field above (suspendedOrRestricted) is only visible in the XFA form when anythingToDeclare is set to "something". When set to "nothing", the declaration text area is hidden. In our package, you can always read and write the field regardless of the radio value — the conditional visibility only applies within Adobe Reader.

This section has no repeatable items. All fields are standalone.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Probity and Health
$data = XfaPdf::read('Section_13_Probity');

// Or using the fluent accessor
$data = XfaPdf::probity()->read();

// $data looks like:
// [
//     'probityChk' => '1',
//     'anythingToDeclare' => 'nothing',
//     'suspendedOrRestricted' => '',
//     'healthChk' => '1',
//     'commentsToAppraiser' => '',
// ]

Read a single field:

$probityConfirmed = XfaPdf::probity()->readField('probityChk');
// '1' (checked) or '0' (unchecked)

$declaration = XfaPdf::probity()->readField('anythingToDeclare');
// 'nothing' or 'something'

Checkbox Fields

The two checkbox fields use "1" for checked and "0" for unchecked. These are confirmation checkboxes where the doctor acknowledges they have read and understood the probity and health statements.

Probity confirmation

Field: probityChk

The doctor confirms they understand and comply with the probity requirements set out by the GMC. Set to "1" for checked or "0" for unchecked.

// Check the probity confirmation
XfaPdf::probity()->updateField('probityChk', '1');

// Uncheck the probity confirmation
XfaPdf::probity()->updateField('probityChk', '0');

Health confirmation

Field: healthChk

The doctor confirms they understand and comply with the health requirements. They acknowledge they have no health conditions that could impair their fitness to practise, or that any conditions are being appropriately managed.

// Check the health confirmation
XfaPdf::probity()->updateField('healthChk', '1');

// Uncheck the health confirmation
XfaPdf::probity()->updateField('healthChk', '0');
Checkbox values: Always use the string "1" for checked and "0" for unchecked. Do not use boolean true/false or integer values — XFA datasets store all values as text nodes in the XML.

Radio / Conditional Fields

The anythingToDeclare field is a radio button group that controls the visibility of the suspendedOrRestricted text area. The radio accepts two values: "nothing" and "something".

Anything to declare?

Field: anythingToDeclare

This radio group asks whether the doctor has any probity concerns to declare. Set it to "nothing" or "something".

// No concerns to declare
XfaPdf::probity()->updateField(
    'anythingToDeclare',
    'nothing'
);

// Has something to declare
XfaPdf::probity()->updateField(
    'anythingToDeclare',
    'something'
);
Important: The radio values are "nothing" and "something" (not "yes"/"no"). Setting anythingToDeclare to "something" in the datasets XML does not automatically make the conditional field visible in the saved PDF — XFA visibility is controlled by JavaScript scripts within the template. However, when Adobe Reader opens the PDF and processes the XFA layer, the radio value will trigger the script and the suspendedOrRestricted field will appear with whatever data you have written into it. Always populate the conditional field when setting the radio to "something".

Setting the radio and conditional field together

When the doctor has something to declare, set the radio and the declaration text in a single call:

// When there IS something to declare, set the radio + declaration text
XfaPdf::probity()->update([
    'anythingToDeclare'      => 'something',
    'suspendedOrRestricted'  => 'I was subject to a conditions of practice '
        . 'undertaking by the GMC from January to June 2024, relating to '
        . 'record-keeping standards. The undertaking has since been lifted '
        . 'and all conditions have been satisfactorily met.',
]);

// When there is NOTHING to declare, just set the radio
XfaPdf::probity()->update([
    'anythingToDeclare' => 'nothing',
]);

Declaration details

Field: suspendedOrRestricted

This text area is conditionally visible when anythingToDeclare is set to "something". The doctor provides details about any suspension, restriction, or other probity concern. This is a multiline text field.

XfaPdf::probity()->updateField(
    'suspendedOrRestricted',
    'I was subject to a conditions of practice undertaking by the GMC '
    . 'from January to June 2024, relating to record-keeping standards. '
    . 'The undertaking has since been lifted and all conditions have been '
    . 'satisfactorily met. Full details were provided to my Responsible '
    . 'Officer at the time.'
);

Text Fields

Comments to appraiser

Field: commentsToAppraiser

A free-text area where the doctor can provide any additional comments to their appraiser regarding probity or health matters. This field is always visible regardless of the radio selection.

XfaPdf::probity()->updateField(
    'commentsToAppraiser',
    'I have no probity or health concerns to raise this year. '
    . 'I continue to maintain appropriate professional boundaries '
    . 'and comply with GMC guidance on Good Medical Practice. '
    . 'I had my annual occupational health review in March 2024 '
    . 'with no issues identified.'
);

Complete Controller Example

Here's a full example filling the entire Probity and Health section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillProbity(Request $request)
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        XfaPdf::setFile($pdfPath);

        // -----------------------------------------------
        // 1. Set the probity confirmation checkbox
        // -----------------------------------------------
        XfaPdf::probity()->updateField('probityChk', '1');

        // -----------------------------------------------
        // 2. Check if there is anything to declare and
        //    fill conditional fields accordingly
        // -----------------------------------------------
        $hasSomethingToDeclare = $request->input('has_declaration', false);

        if ($hasSomethingToDeclare) {
            // Doctor has something to declare: set radio
            // to "something" and populate the details field
            XfaPdf::probity()->update([
                'anythingToDeclare'     => 'something',
                'suspendedOrRestricted' => $request->input(
                    'declaration_details',
                    ''
                ),
            ]);
        } else {
            // Nothing to declare: just set the radio
            XfaPdf::probity()->updateField(
                'anythingToDeclare',
                'nothing'
            );
        }

        // -----------------------------------------------
        // 3. Set the health confirmation checkbox
        // -----------------------------------------------
        XfaPdf::probity()->updateField('healthChk', '1');

        // -----------------------------------------------
        // 4. Add any comments to the appraiser
        // -----------------------------------------------
        $comments = $request->input('comments_to_appraiser', '');
        if ($comments) {
            XfaPdf::probity()->updateField(
                'commentsToAppraiser',
                $comments
            );
        }

        // -----------------------------------------------
        // 5. Save the PDF
        // -----------------------------------------------
        XfaPdf::save();

        return redirect()->back()->with(
            'success',
            'Probity and Health saved.'
        );
    }

    /**
     * Read and display existing Probity and Health data.
     */
    public function showProbity()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        XfaPdf::setFile($pdfPath);

        $data = XfaPdf::probity()->read();

        // Check if conditional field should be shown
        $hasDeclaration = ($data['anythingToDeclare'] ?? '') === 'something';

        return view('appraisal.probity', [
            'probityConfirmed'      => ($data['probityChk'] ?? '0') === '1',
            'hasDeclaration'        => $hasDeclaration,
            'suspendedOrRestricted' => $data['suspendedOrRestricted'] ?? '',
            'healthConfirmed'       => ($data['healthChk'] ?? '0') === '1',
            'commentsToAppraiser'   => $data['commentsToAppraiser'] ?? '',
        ]);
    }
}
