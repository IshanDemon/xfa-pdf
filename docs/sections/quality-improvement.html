Section 8: Quality Improvement Activity

This section captures the doctor's quality improvement activities such as clinical audits, case reviews, and service improvement projects. It includes a commentary field and a repeatable table of activities.

Section name: Section_8_QualityImprovement
Fluent accessor: XfaPdf::qualityImprovement()

What's in this section?

FieldTypeWhat it is
evidenceOfParticipationMultiline textCommentary on QI activities and how they demonstrate Good Medical Practice
appriaserCommentsMultiline textAppraiser comments on QI activities (note: field name contains original typo)
improvementActivities > activityRepeatableQI activity rows, each with role, description, outcome, and supporting details

Each activity row inside improvementActivities contains: role (select), description (textarea), outcome (textarea), supportLocation (select), firstEntry (text), amendData (text).

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Quality Improvement
$data = XfaPdf::read('Section_8_QualityImprovement');

// Or using the fluent accessor
$data = XfaPdf::qualityImprovement()->read();

// $data looks like:

Quality Improvement Activities (Repeatable Table)

Each row in the quality improvement table represents one QI activity. You can add multiple rows, remove them, or replace all of them. Each row contains the following fields:

Row fieldTypeDescription
roleSelectThe doctor's role in the activity (e.g. Lead, Participant)
descriptionTextareaDescription of the quality improvement activity
outcomeTextareaOutcomes and learning from the activity
supportLocationSelectWhere supporting evidence is held (e.g. Portfolio, Attached)
firstEntryTextDate the activity was first recorded
amendDataTextDate of any subsequent amendments

Adding a new activity row

To add a new QI activity row:

$manager = app('xfa-pdf');
$xfaPdf = $manager->load(storage_path('app/xfa-pdf/appraisal.pdf'));

// Add a quality improvement activity
$xfaPdf = $manager->addRepeatableItem(
    $xfaPdf,
    'Section_8_QualityImprovement',    // section name
    'improvementActivities',            // container name
    [
        'role'            => 'Lead',
        'description'     => 'Clinical audit of diabetes management outcomes across the practice. '
                           . 'Reviewed HbA1c targets, medication adherence, and annual review rates '
                           . 'for all Type 2 diabetes patients over a 12-month period.',
        'outcome'         => 'Identified that 15% of patients had not received an annual review. '
                           . 'Implemented a recall system which improved review rates from 78% to 94%. '
                           . 'HbA1c control improved in 23% of patients reviewed.',
        'supportLocation' => 'Portfolio',
        'firstEntry'      => '2024-03-15',
        'amendData'       => '',
    ]
);

$manager->saveXfaPdf($xfaPdf);

Reading existing activity rows

$activities = $manager->getRepeatableItems(
    $xfaPdf,
    'Section_8_QualityImprovement',
    'improvementActivities'
);

foreach ($activities as $index => $activity) {
    echo "Activity {$index}: [{$activity['role']}] {$activity['description']}\n";
    echo "  Outcome: {$activity['outcome']}\n";
}
// Activity 0: [Lead] Clinical audit of diabetes management outcomes...
//   Outcome: Annual review rates improved from 78% to 94%...
// Activity 1: [Participant] Review of emergency admission pathways...
//   Outcome: Developed a community-based falls assessment pathway...
// Activity 2: [Lead] Service improvement project: streamlining repeat prescription requests...
//   Outcome: Reduced average processing time from 72 hours to 24 hours...

Removing an activity row

Remove a row by its index (0-based):

// Remove the second activity (index 1)
$xfaPdf = $manager->removeRepeatableItem(
    $xfaPdf,
    'Section_8_QualityImprovement',
    'improvementActivities',
    1
);

$manager->saveXfaPdf($xfaPdf);

Text Fields

In addition to the repeatable activity table, this section has two standalone text fields for commentary.

Evidence of participation

Field: evidenceOfParticipation

This is where the doctor provides an overall commentary on their quality improvement activities and how they demonstrate Good Medical Practice. The form label reads: "Please provide commentary on your quality improvement activities."

XfaPdf::qualityImprovement()->updateField(
    'evidenceOfParticipation',
    'I have undertaken three quality improvement activities this year, covering '
    . 'clinical audit, multi-disciplinary pathway redesign, and service improvement. '
    . 'The diabetes audit (Domain 1: Knowledge) demonstrated my commitment to '
    . 'evidence-based practice and resulted in measurable improvements in patient '
    . 'outcomes. The falls pathway project (Domain 2: Safety) involved working '
    . 'across organisational boundaries to reduce avoidable hospital admissions. '
    . 'The prescription workflow project (Domain 3: Communication) improved '
    . 'patient experience and practice efficiency.'
);

Appraiser's comments

Field: appriaserComments

Note: The field name appriaserComments contains a typo (missing 'p' in "appraiser"). This matches the actual field name in the MAG42 dataset XML. Use the exact spelling shown here.

Written by the appraiser, summarising their view of the doctor's quality improvement activities.

XfaPdf::qualityImprovement()->updateField(
    'appriaserComments',
    'Dr Smith has demonstrated a strong commitment to quality improvement this year. '
    . 'The diabetes audit is a well-designed clinical audit with a clear re-audit cycle '
    . 'showing sustained improvement. The falls pathway project is an excellent example '
    . 'of cross-organisational QI work with measurable patient benefit. I would encourage '
    . 'Dr Smith to consider publishing the prescription workflow findings as a case study '
    . 'for the wider primary care community.'
);

Complete Controller Example

Here's a full example filling the entire Quality Improvement section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillQualityImprovement()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        $manager = app('xfa-pdf');
        $xfaPdf = $manager->load($pdfPath);

        // -----------------------------------------------
        // 1. Set quality improvement activity rows
        // -----------------------------------------------
        $xfaPdf = $manager->setRepeatableItems(
            $xfaPdf,
            'Section_8_QualityImprovement',
            'improvementActivities',
            'activity',
            [
                [
                    'role'            => 'Lead',
                    'description'     => 'Clinical audit of diabetes management '
                                       . 'outcomes. Reviewed HbA1c targets, medication '
                                       . 'adherence, and annual review rates for 840 '
                                       . 'Type 2 diabetes patients over 12 months.',
                    'outcome'         => 'Annual review rates improved from 78% to 94% '
                                       . 'after implementing a structured recall system. '
                                       . 'HbA1c control improved in 23% of patients. '
                                       . 'Re-audit planned for Q2 2025.',
                    'supportLocation' => 'Portfolio',
                    'firstEntry'      => '2024-03-15',
                    'amendData'       => '',
                ],
                [
                    'role'            => 'Participant',
                    'description'     => 'Review of emergency admission pathways for '
                                       . 'elderly patients presenting with falls. '
                                       . 'Multi-disciplinary project with A&E, '
                                       . 'physiotherapy, and community nursing.',
                    'outcome'         => 'Developed a community-based falls assessment '
                                       . 'pathway. Reduced unnecessary hospital '
                                       . 'admissions by 12% over 6 months. Findings '
                                       . 'presented at the Regional QI Conference.',
                    'supportLocation' => 'Portfolio',
                    'firstEntry'      => '2024-07-01',
                    'amendData'       => '',
                ],
                [
                    'role'            => 'Lead',
                    'description'     => 'Service improvement project: streamlining '
                                       . 'repeat prescription requests. Audited '
                                       . 'turnaround times and identified bottlenecks '
                                       . 'in the administrative workflow.',
                    'outcome'         => 'Reduced average processing time from 72 hours '
                                       . 'to 24 hours. Patient complaints regarding '
                                       . 'prescription delays decreased by 60%. Now '
                                       . 'adopted as standard practice across the PCN.',
                    'supportLocation' => 'Attached',
                    'firstEntry'      => '2024-09-20',
                    'amendData'       => '2024-11-05',
                ],
            ]
        );

        // -----------------------------------------------
        // 2. Fill text fields
        // -----------------------------------------------
        XfaPdf::setFile($pdfPath);

        XfaPdf::qualityImprovement()->update([
            'evidenceOfParticipation' =>
                'I have undertaken three quality improvement activities this '
                . 'year, covering clinical audit, multi-disciplinary pathway '
                . 'redesign, and service improvement. The diabetes audit '
                . '(Domain 1: Knowledge) demonstrated my commitment to '
                . 'evidence-based practice and resulted in measurable '
                . 'improvements in patient outcomes. The falls pathway '
                . 'project (Domain 2: Safety) involved working across '
                . 'organisational boundaries to reduce avoidable hospital '
                . 'admissions. The prescription workflow project (Domain 3: '
                . 'Communication) improved patient experience and efficiency.',

            'appriaserComments' =>
                'Dr Smith has demonstrated a strong commitment to quality '
                . 'improvement. The diabetes audit is well-designed with a '
                . 'clear re-audit cycle. The falls pathway project is an '
                . 'excellent example of cross-organisational QI work. I would '
                . 'encourage publishing the prescription workflow findings '
                . 'as a case study for the wider primary care community.',
        ]);

        // -----------------------------------------------
        // 3. Save the PDF
        // -----------------------------------------------
        $manager->saveXfaPdf($xfaPdf);
        // Or: XfaPdf::save();

        return redirect()->back()->with('success', 'Quality Improvement saved.');
    }
}
