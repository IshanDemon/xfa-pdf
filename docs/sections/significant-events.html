Section 9: Significant Events

This section records significant events the doctor has been involved in, such as serious incidents, near misses, or clinical errors. The doctor declares whether they have been involved in any significant events and provides details if so.

Section name: Section_9_SignificantEvents
Fluent accessor: XfaPdf::significantEvents()

What's in this section?

FieldTypeWhat it is
haveBeenInvolvedRadio (not/have)Whether the doctor has been involved in significant events (conditional trigger)
significantAttachments > attachRepeatable rowsIndividual significant event entries (shown when "have")
  roleSelectDoctor's role in the event
  descriptionTextareaDescription of the significant event
  outcomeTextareaOutcome and actions taken
  supportLocationSelectWhere the supporting evidence is located
  firstEntryTextDate of first entry
  amendDataTextDate of last amendment
commentaryTextareaOverall commentary on significant events (shown when "have")
appraiserCommentsTextareaAppraiser's comments (mirrors to Section 19)
Conditional fields: The haveBeenInvolved radio uses the values "not" and "have" — not "yes"/"no". When set to "have", the repeatable event table (attach) and the commentary field become visible. When set to "not", both are hidden.

Setup

Before filling any fields, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Significant Events
$data = XfaPdf::read('Section_9_SignificantEvents');

// Or using the fluent accessor
$data = XfaPdf::significantEvents()->read();

// $data looks like:
// [
//     'haveBeenInvolved' => 'have',
//     'significantAttachments' => [
//         'attach' => [
//             [
//                 'role' => 'Lead clinician',
//                 'description' => 'Delayed diagnosis of sepsis...',
//                 'outcome' => 'Root cause analysis completed...',
//                 'supportLocation' => 'Attached',
//                 'firstEntry' => '2024-03-15',
//                 'amendData' => '2024-06-20',
//             ],
//         ],
//     ],
//     'commentary' => 'I was involved in one significant event...',
//     'appraiserComments' => '',
// ]

Read a single field:

$involved = XfaPdf::significantEvents()->readField('haveBeenInvolved');
// 'have' or 'not'

Radio / Conditional Fields

The haveBeenInvolved field is a radio group that controls whether the event table and commentary are shown. It accepts two values:

ValueMeaningEffect
"not"Not been involved in any significant eventsHides the event table and commentary field
"have"Have been involved in significant eventsShows the event table and commentary field

Setting the radio value

When the doctor has been involved in significant events:

// Doctor has been involved in significant events
XfaPdf::significantEvents()->updateField('haveBeenInvolved', 'have');

When the doctor has not been involved:

// Doctor has NOT been involved in significant events
XfaPdf::significantEvents()->updateField('haveBeenInvolved', 'not');
Important: If you set haveBeenInvolved to "have", you should also populate at least one event row in the attach repeatable table. If set to "not", the event rows and commentary will be hidden in the rendered XFA form regardless of their content.

Significant Events (Repeatable Table)

Each row in the significant events table represents one event. Rows live inside significantAttachments and are named attach. Each row contains six fields:

FieldTypeWhat to put here
roleSelectThe doctor's role in the event (e.g. "Lead clinician", "Team member")
descriptionTextareaWhat happened, including the nature and severity of the event
outcomeTextareaThe outcome, investigation findings, and actions taken
supportLocationSelectWhere the supporting evidence is filed (e.g. "Attached", "Held locally")
firstEntryTextDate the event was first recorded
amendDataTextDate of the most recent amendment

Adding a new event row

To add a single significant event:

$manager = app('xfa-pdf');
$xfaPdf = $manager->load(storage_path('app/xfa-pdf/appraisal.pdf'));

// Add a significant event row
$xfaPdf = $manager->addRepeatableItem(
    $xfaPdf,
    'Section_9_SignificantEvents',   // section name
    'significantAttachments',         // container name
    [
        'role'            => 'Lead clinician',
        'description'     => 'Delayed diagnosis of sepsis in a 72-year-old patient presenting '
                           . 'with non-specific symptoms. Initial assessment did not identify '
                           . 'early warning signs. Patient deteriorated over 6 hours before '
                           . 'escalation to acute medical team.',
        'outcome'         => 'Root cause analysis completed. Patient recovered fully after '
                           . 'ICU admission. New sepsis screening protocol implemented in '
                           . 'practice. All clinical staff completed sepsis awareness training.',
        'supportLocation' => 'Attached',
        'firstEntry'      => '2024-03-15',
        'amendData'       => '2024-06-20',
    ]
);

$manager->saveXfaPdf($xfaPdf);

Adding multiple events

If you need to add several events, call addRepeatableItem() for each one:

$manager = app('xfa-pdf');
$xfaPdf = $manager->load(storage_path('app/xfa-pdf/appraisal.pdf'));

// Event 1: Delayed diagnosis
$xfaPdf = $manager->addRepeatableItem($xfaPdf, 'Section_9_SignificantEvents', 'significantAttachments', [
    'role'            => 'Lead clinician',
    'description'     => 'Delayed diagnosis of sepsis in a 72-year-old patient. '
                       . 'Initial assessment missed early warning signs.',
    'outcome'         => 'Root cause analysis completed. New sepsis screening '
                       . 'protocol implemented. All staff completed training.',
    'supportLocation' => 'Attached',
    'firstEntry'      => '2024-03-15',
    'amendData'       => '2024-06-20',
]);

// Event 2: Prescribing error
$xfaPdf = $manager->addRepeatableItem($xfaPdf, 'Section_9_SignificantEvents', 'significantAttachments', [
    'role'            => 'Prescriber',
    'description'     => 'Prescribing error involving methotrexate. Weekly dose '
                       . 'was incorrectly entered as daily on repeat prescription. '
                       . 'Error caught by community pharmacist before dispensing.',
    'outcome'         => 'Near miss reported via Datix. Prescription corrected '
                       . 'immediately. Practice prescribing protocol updated to '
                       . 'include mandatory high-risk drug review at each reauthorisation.',
    'supportLocation' => 'Held locally',
    'firstEntry'      => '2024-07-02',
    'amendData'       => '2024-07-10',
]);

// Event 3: Safeguarding concern
$xfaPdf = $manager->addRepeatableItem($xfaPdf, 'Section_9_SignificantEvents', 'significantAttachments', [
    'role'            => 'Team member',
    'description'     => 'Safeguarding referral for a vulnerable adult following '
                       . 'home visit. Patient showed signs of neglect and possible '
                       . 'financial exploitation by a family member.',
    'outcome'         => 'Multi-agency safeguarding meeting held. Social services '
                       . 'involved and care plan established. Regular review visits '
                       . 'scheduled. Case discussed at practice MDT.',
    'supportLocation' => 'Attached',
    'firstEntry'      => '2024-09-18',
    'amendData'       => '2024-11-05',
]);

$manager->saveXfaPdf($xfaPdf);

Replacing all event rows

If you want to clear existing rows and set new ones from scratch, use setRepeatableItems():

$xfaPdf = $manager->setRepeatableItems(
    $xfaPdf,
    'Section_9_SignificantEvents',   // section
    'significantAttachments',         // container
    'attach',                         // element name
    [
        [
            'role'            => 'Lead clinician',
            'description'     => 'Delayed diagnosis of sepsis in elderly patient.',
            'outcome'         => 'Root cause analysis completed. Sepsis protocol updated.',
            'supportLocation' => 'Attached',
            'firstEntry'      => '2024-03-15',
            'amendData'       => '2024-06-20',
        ],
        [
            'role'            => 'Prescriber',
            'description'     => 'Methotrexate prescribing error caught by pharmacist.',
            'outcome'         => 'Near miss reported. Prescribing protocol updated.',
            'supportLocation' => 'Held locally',
            'firstEntry'      => '2024-07-02',
            'amendData'       => '2024-07-10',
        ],
    ]
);

$manager->saveXfaPdf($xfaPdf);

Reading existing event rows

$rows = $manager->getRepeatableItems(
    $xfaPdf,
    'Section_9_SignificantEvents',
    'significantAttachments'
);

foreach ($rows as $index => $row) {
    echo "Event {$index}: {$row['role']} - {$row['description']}\n";
}
// Event 0: Lead clinician - Delayed diagnosis of sepsis in elderly patient.
// Event 1: Prescriber - Methotrexate prescribing error caught by pharmacist.

Removing an event row

Remove an event by its index (0-based):

// Remove the second event (index 1)
$xfaPdf = $manager->removeRepeatableItem(
    $xfaPdf,
    'Section_9_SignificantEvents',
    'significantAttachments',
    1
);

$manager->saveXfaPdf($xfaPdf);

Text Fields

Below the event table, there are two free-text fields. The commentary field is conditionally shown (only visible when haveBeenInvolved is "have"), while appraiserComments is always available.

Commentary on significant events

Field: commentary

This is where the doctor provides an overall reflection on their significant events, including what they have learned and how their practice has changed as a result. Only shown when haveBeenInvolved is "have".

XfaPdf::significantEvents()->updateField(
    'commentary',
    'I was involved in two significant events during this appraisal period. '
    . 'The delayed sepsis diagnosis prompted a thorough review of our acute '
    . 'assessment processes and led to the implementation of a standardised '
    . 'sepsis screening tool. The prescribing near miss reinforced the '
    . 'importance of robust repeat prescribing systems for high-risk drugs. '
    . 'Both events were discussed openly at practice meetings and the learning '
    . 'has been shared across the primary care network.'
);

Appraiser's comments

Field: appraiserComments

Written by the appraiser, summarising their view of the doctor's significant events and reflective practice. This field mirrors to Section 19 (Appraisal Summary).

XfaPdf::significantEvents()->updateField(
    'appraiserComments',
    'Dr Smith has demonstrated excellent reflective practice following two '
    . 'significant events. The sepsis case has led to meaningful systemic '
    . 'change within the practice, and the prescribing near miss was handled '
    . 'proactively with appropriate protocol changes. Both events show a '
    . 'commitment to patient safety and continuous improvement.'
);

Complete Controller Example

Here's a full example filling the entire Significant Events section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillSignificantEvents()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        $manager = app('xfa-pdf');
        $xfaPdf = $manager->load($pdfPath);

        // -----------------------------------------------
        // 1. Set the conditional radio
        // -----------------------------------------------
        XfaPdf::setFile($pdfPath);
        XfaPdf::significantEvents()->updateField('haveBeenInvolved', 'have');

        // -----------------------------------------------
        // 2. Add significant event rows
        // -----------------------------------------------
        $xfaPdf = $manager->setRepeatableItems(
            $xfaPdf,
            'Section_9_SignificantEvents',
            'significantAttachments',
            'attach',
            [
                [
                    'role'            => 'Lead clinician',
                    'description'     => 'Delayed diagnosis of sepsis in a '
                                       . '72-year-old patient presenting with '
                                       . 'non-specific symptoms. Patient '
                                       . 'deteriorated over 6 hours before '
                                       . 'escalation to acute medical team.',
                    'outcome'         => 'Root cause analysis completed. Patient '
                                       . 'recovered fully after ICU admission. '
                                       . 'New sepsis screening protocol '
                                       . 'implemented. All clinical staff '
                                       . 'completed sepsis awareness training.',
                    'supportLocation' => 'Attached',
                    'firstEntry'      => '2024-03-15',
                    'amendData'       => '2024-06-20',
                ],
                [
                    'role'            => 'Prescriber',
                    'description'     => 'Prescribing error involving methotrexate. '
                                       . 'Weekly dose incorrectly entered as daily '
                                       . 'on repeat prescription. Caught by '
                                       . 'community pharmacist before dispensing.',
                    'outcome'         => 'Near miss reported via Datix. Prescription '
                                       . 'corrected immediately. High-risk drug '
                                       . 'review now mandatory at each repeat '
                                       . 'reauthorisation.',
                    'supportLocation' => 'Held locally',
                    'firstEntry'      => '2024-07-02',
                    'amendData'       => '2024-07-10',
                ],
            ]
        );

        // -----------------------------------------------
        // 3. Fill text fields
        // -----------------------------------------------
        XfaPdf::significantEvents()->update([
            'commentary' =>
                'I was involved in two significant events this year. '
                . 'The delayed sepsis diagnosis prompted a thorough review '
                . 'of our acute assessment processes and led to a new '
                . 'standardised screening tool. The prescribing near miss '
                . 'reinforced the importance of robust repeat prescribing '
                . 'systems. Both events were discussed at practice meetings '
                . 'and learning shared across the PCN.',

            'appraiserComments' =>
                'Dr Smith has demonstrated excellent reflective practice. '
                . 'The sepsis case led to meaningful systemic change, and '
                . 'the prescribing near miss was handled proactively. Both '
                . 'events show a commitment to patient safety and continuous '
                . 'improvement.',
        ]);

        // -----------------------------------------------
        // 4. Save the PDF
        // -----------------------------------------------
        $manager->saveXfaPdf($xfaPdf);
        // Or: XfaPdf::save();

        return redirect()->back()->with('success', 'Significant Events saved.');
    }
}
