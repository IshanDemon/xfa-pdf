Section 15: Supporting Information

A minimal section that tracks the total size of supporting information attachments. This is one of the simplest sections in the form — it contains only a single numeric field with no repeatable items or conditional logic.

Section name: Section_15_SupportingInformation
Fluent accessor: XfaPdf::supportingInformation()
Navigation label: Supporting information

What's in this section?

FieldTypeWhat it is
attachmentTotalSizeNumberTotal size of all supporting information attachments

This section has no repeatable items and no conditional fields. It contains only a single standalone numeric field.

Setup

Before reading or writing this field, load the PDF in your controller:

use Xfa\Pdf\Facades\XfaPdf;

// Load the PDF file
XfaPdf::setFile(storage_path('app/xfa-pdf/appraisal.pdf'));

Reading existing data

Read all fields in this section at once:

// Read everything in Supporting Information
$data = XfaPdf::read('Section_15_SupportingInformation');

// Or using the fluent accessor
$data = XfaPdf::supportingInformation()->read();

// $data looks like:
// [
//     'attachmentTotalSize' => '2048',
// ]

Read the single field directly:

$totalSize = XfaPdf::supportingInformation()->readField('attachmentTotalSize');
// '2048'

Updating the Field

The attachmentTotalSize field is a numeric value representing the total size of all supporting information attachments. In the XFA form this uses a numericEdit control. Pass the value as a string (all XFA field values are stored as strings in the datasets XML).

Attachment total size

Field: attachmentTotalSize

Set the total size of all supporting information attachments. The value is typically calculated by the XFA form itself as attachments are added, but you can also set it programmatically.

// Set the total attachment size
XfaPdf::supportingInformation()->updateField(
    'attachmentTotalSize',
    '4096'
);

// Save the PDF
XfaPdf::save();

You can also use the update() method with an array, though with only one field the difference is minimal:

XfaPdf::supportingInformation()->update([
    'attachmentTotalSize' => '4096',
]);

XfaPdf::save();
Note: Although this is a numeric field, always pass the value as a string. All XFA dataset values are stored as text nodes in the XML. The XFA renderer in Adobe Reader handles numeric formatting and validation based on the template definition.

Complete Controller Example

Here's a full example reading and updating the Supporting Information section in a Laravel controller:

use Xfa\Pdf\Facades\XfaPdf;

class AppraisalController extends Controller
{
    public function fillSupportingInformation(Request $request)
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        XfaPdf::setFile($pdfPath);

        // -----------------------------------------------
        // 1. Calculate total attachment size from uploaded
        //    files or stored records
        // -----------------------------------------------
        $totalBytes = $request->input('total_attachment_size', 0);

        // -----------------------------------------------
        // 2. Update the field
        // -----------------------------------------------
        XfaPdf::supportingInformation()->updateField(
            'attachmentTotalSize',
            (string) $totalBytes
        );

        // -----------------------------------------------
        // 3. Save the PDF
        // -----------------------------------------------
        XfaPdf::save();

        return redirect()->back()->with(
            'success',
            'Supporting Information saved.'
        );
    }

    /**
     * Read and display existing Supporting Information data.
     */
    public function showSupportingInformation()
    {
        $pdfPath = storage_path('app/xfa-pdf/appraisal.pdf');
        XfaPdf::setFile($pdfPath);

        $data = XfaPdf::supportingInformation()->read();

        return view('appraisal.supporting-information', [
            'attachmentTotalSize' => $data['attachmentTotalSize'] ?? '0',
        ]);
    }
}
