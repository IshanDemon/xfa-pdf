Troubleshooting

"The file is damaged and could not be repaired"

This Adobe Reader error means the incremental update has a structural problem. Common causes:

  • Xref table offsets are incorrect — byte positions must be exact
  • New stream object has wrong /Length — must match exact byte length of new XML
  • Trailer /Prev doesn't point to previous xref offset
  • Object number doesn't match the datasets stream from the /XFA array

PDF opens but shows blank pages in Chrome/Firefox/Safari

This is expected behavior. XFA PDFs are only rendered by Adobe Acrobat Reader. Browser PDF viewers do not support XFA. Static XFA forms may show an AcroForm fallback, but dynamic XFA forms will show blank pages. Always open XFA PDFs in Adobe Acrobat Reader.

Fields not saving / values disappearing after save

Check these:

  • Field path is incorrect — use XfaPdf::sections() and XfaPdf::read() to verify exact names
  • Namespace mismatch — verify NamespaceService::detect() finds the correct data namespace
  • Template being modified — only datasets XML should be changed, never the template
  • Wrong object number — the datasets stream object number must match the /XFA array reference

NoXfaDataException: No datasets found

The PDF doesn't contain XFA data streams:

  • The PDF may be a standard AcroForm, not XFA — this package only handles XFA
  • Streams may use unsupported compression — the package supports Flate (zlib) only
  • The /XFA array reference may be missing from the AcroForm dictionary

Namespace detection returning wrong prefix

The NamespaceService auto-detects namespaces from the XML root element. If detection seems wrong, inspect the XML with XfaPdf::rawXml(). The service uses local-name() in XPath for namespace-agnostic field matching, so most queries work regardless of prefix.

Upload fails with "file too large"

Two things to check:

  • Increase max_upload_size in config/xfa-pdf.php (default: 20 MB)
  • Check PHP settings: upload_max_filesize and post_max_size in php.ini

Preview shows wrong field types

The preview renders fields based on template metadata. If a field shows as a text input when it should be a dropdown, ensure the template XML is being extracted correctly. Check XfaPdf::fieldMetadata() to see what types are detected.

Repeatable items not appearing after add

Verify:

  • The section name and container name are correct
  • The container exists in the datasets XML (check with XfaPdf::read())
  • The CSRF token is being sent with AJAX requests
  • Check FieldNotFoundException::containerNotFound() is not being thrown
