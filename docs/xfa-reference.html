XFA Technical Reference

Background on the XFA format for developers working with this package.

What is XFA?

XFA (XML Forms Architecture) is an Adobe XML-based PDF format where form structure (template) and data (datasets) are stored as separate XML packets inside a PDF container. Unlike standard AcroForms, XFA supports dynamic content reflow, growable fields, and repeating sections.

Important: XFA is deprecated in PDF 2.0. Only Adobe Acrobat/Reader fully renders XFA forms. Browser PDF viewers do NOT support XFA.

PDF Structure

An XFA PDF is a standard PDF shell with XFA XML referenced from the AcroForm dictionary:

PDF Catalog -> /AcroForm -> /XFA [
    (preamble)  streamRef    // Opening <xdp:xdp> tag
    (config)    streamRef    // Processing instructions
    (template)  streamRef    // Form layout & field definitions
    (datasets)  streamRef    // Form field data values
    (postamble) streamRef    // Closing </xdp:xdp> tag
]

XDP Wrapper

<?xml version="1.0" encoding="UTF-8"?>
<xdp:xdp xmlns:xdp="http://ns.adobe.com/xdp/">
    <config xmlns="http://www.xfa.org/schema/xci/3.0/">...</config>
    <template xmlns="http://www.xfa.org/schema/xfa-template/3.3/">...</template>
    <xfa:datasets xmlns:xfa="http://www.xfa.org/schema/xfa-data/1.0/">
        <xfa:data>...</xfa:data>
    </xfa:datasets>
</xdp:xdp>

Data Binding

Fields bind to dataset nodes by name hierarchy. A field firstName in subform page1 maps to:

<xfa:datasets xmlns:xfa="http://www.xfa.org/schema/xfa-data/1.0/">
  <xfa:data>
    <form1>
      <page1>
        <firstName>John</firstName>
      </page1>
    </form1>
  </xfa:data>
</xfa:datasets>

Binding modes:

  • match="dataRef" — default, binds by name hierarchy
  • match="global" — searches entire data tree
  • match="none" — no data binding

Incremental PDF Updates

This package uses incremental updates to save changes, the same mechanism Adobe Reader uses:

  • New object versions are appended at the end of the file
  • A new xref (cross-reference) table points to the updated objects
  • Original PDF content is completely untouched
  • All interactive features (scripts, signatures) are preserved
  • Much faster than rewriting the entire file
